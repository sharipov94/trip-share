# Tutorial: запускаем TravelMate с нуля

За этот туториал ты поднимешь работающий прототип TravelMate на своей машине:
сначала кликабельный фронт на заглушках (за 3 шага, без бэкенда), потом — полный
стек с реальным API и базой. К концу ты будешь понимать, как устроен запуск обоих
пакетов и где переключатель MOCK/реальный backend.

## Что понадобится

- **Node.js 22+** и **npm** (`node -v` должно показать v22 или выше)
- **Docker** — только для второй части (база данных)
- Терминал и браузер

Telegram для прохождения туториала **не нужен** — есть dev-вход.

---

## Часть 1. Кликабельный фронт за 3 шага (MOCK)

### Шаг 1. Установи зависимости фронта

```bash
cd app
npm install
```

Скачиваются пакеты Vite, React, TanStack Query. Один раз на машину.

### Шаг 2. Запусти дев-сервер

```bash
npm run dev
```

В консоли появится строка вида `Local: http://localhost:5173/`. Vite поднял
сервер с горячей перезагрузкой.

### Шаг 3. Открой приложение

Открой `http://localhost:5173` в браузере. **Готово** — ты видишь рабочий прототип:
нижняя навигация (Сегодня · Поездки · Расходы · Фото · Профиль), мок-поездки
вроде «Барселона 2027», почти всё кликается.

Почему это сработало без бэкенда: `VITE_API_URL` пустой, поэтому
`app/src/api/client.ts` включил MOCK-режим и берёт данные из `src/mocks/data.ts`.

Попробуй: зайди в **Профиль** и переключи палитру — тема меняется мгновенно и
сохраняется в `localStorage`.

## Часть 2. Полный стек с реальным API

Теперь поднимем бэкенд и подключим к нему фронт.

### Шаг 4. Подними базу и сервер

В новом терминале:

```bash
cd server
cp .env.example .env
docker compose up -d
```

`docker compose` поднял PostgreSQL на `:5432`. Теперь сам сервер:

```bash
npm install
npm run start:dev
```

Когда увидишь `TravelMate API → http://localhost:3000/api/v1` — API живой.
Таблицы создались автоматически из entities (`DB_SYNCHRONIZE=true`).

### Шаг 5. Залогинься без Telegram

Реальный Telegram `initData` локально не получить, поэтому используем dev-вход
(он включён через `DEV_FAKE_AUTH=true` в `.env.example`):

```bash
curl -X POST localhost:3000/api/v1/auth/dev \
  -H 'content-type: application/json' -d '{"firstName":"Никита"}'
```

Ответ содержит `access` и `refresh` токены — значит аутентификация работает.

### Шаг 6. Переключи фронт на реальный backend

В первом терминале останови `npm run dev` (Ctrl+C) и задай адрес API:

```bash
cd app
cp .env.example .env
```

Открой `app/.env` и впиши:

```
VITE_API_URL=http://localhost:3000
```

Запусти снова:

```bash
npm run dev
```

Открой `http://localhost:5173`. Теперь фронт ходит в реальный API: в DevTools →
Network видны запросы к `/api/v1/...`. Приложение само залогинится через dev-вход
(фронт вызывает `/auth/dev`, когда нет Telegram).

---

## Что ты собрал

У тебя на машине крутится полный стек TravelMate:

- **Фронт** (`app/`) на `:5173` — Telegram Mini App в браузере
- **API** (`server/`) на `:3000/api/v1` — NestJS
- **PostgreSQL** в Docker на `:5432`

И ты понял главный переключатель: пустой `VITE_API_URL` → MOCK (заглушки), адрес →
реальный backend. Это делает фронт-разработку независимой от сервера.
