# Access Control — модель доступа

Все эндпоинты (кроме auth) защищены двумя слоями:

1. **Аутентификация** — валидный JWT (см. [07-auth](07-auth.md)).
2. **Авторизация** — guard проверяет членство в поездке и роль.

## Роли (`member_role`)

| Роль | Описание |
|------|----------|
| `owner` | Создатель поездки. Полный контроль, включая удаление поездки. |
| `admin` | Назначается owner'ом. Управляет участниками и контентом поездки. |
| `member` | Обычный участник. Создаёт свой контент, голосует, добавляет расходы. |

Доступ к ресурсу всегда ограничен участниками его поездки: не-член поездки не видит
и не меняет ничего внутри неё.

## Правило владения

Сверх роли действует правило **владения объектом**: автор активности/расхода/
комментария может редактировать и удалять свой объект. `admin`/`owner` могут
модерировать чужой контент.

## Матрица: действие × роль

Легенда: ✅ можно · ⚙️ только владелец объекта · ❌ нельзя.

| Действие | owner | admin | member | автор объекта |
|----------|:-----:|:-----:|:------:|:-------------:|
| Просмотр поездки и контента | ✅ | ✅ | ✅ | — |
| Изменить поездку (`PATCH /trips/:id`) | ✅ | ✅ | ❌ | — |
| Удалить поездку (`DELETE /trips/:id`) | ✅ | ❌ | ❌ | — |
| Создать инвайт | ✅ | ✅ | ❌ | — |
| Удалить участника | ✅ | ✅ | ❌ | — |
| Назначить/снять admin | ✅ | ❌ | ❌ | — |
| Покинуть поездку | ✅¹ | ✅ | ✅ | — |
| Создать активность | ✅ | ✅ | ✅ | — |
| Изменить/удалить активность | ✅ | ✅ | ❌ | ⚙️ |
| Завершить активность (`complete`) | ✅ | ✅ | ❌ | ⚙️ |
| Голосовать | ✅ | ✅ | ✅ | — |
| Комментировать | ✅ | ✅ | ✅ | — |
| Удалить комментарий | ✅ | ✅ | ❌ | ⚙️ |
| Создать расход | ✅ | ✅ | ✅ | — |
| Изменить/удалить расход | ✅ | ✅ | ❌ | ⚙️ (плательщик) |
| Отметить долг оплаченным | ✅ | ✅ | ✅² | — |
| Загрузить фото/чек | ✅ | ✅ | ✅ | — |
| Удалить фото | ✅ | ✅ | ❌ | ⚙️ (автор) |
| Сгенерировать/смотреть Summary | ✅ | ✅ | ✅ | — |

¹ `owner` не может просто покинуть поездку, не передав владение — иначе поездка
останется без owner'а. Сначала передача роли `owner` другому участнику.

² Долг может отметить оплаченным участник-должник (подтверждает, что перевёл) или
кредитор/admin (подтверждает получение) — конкретная политика фиксируется при
реализации; по умолчанию — обе стороны.

## Реализация в NestJS

- `JwtAuthGuard` (глобальный) — извлекает пользователя из токена.
- `TripMemberGuard` — по `:tripId`/`:id` ресурса находит членство, кладёт `role`
  в `request`. Отказ → `403`.
- `@Roles('owner', 'admin')` + `RolesGuard` — для действий, требующих повышенной
  роли.
- Проверка владения — в сервисе (сравнение `creator_id`/`payer_id`/`user_id` с
  текущим пользователем), т.к. зависит от данных объекта.
