# TravelMate / Trip Share — Навигация v2.0 (дизайн-спека)

**Статус:** черновик на ревью · **Дата:** 2026-06-25 · **База:** тег `v1.0`, ветка `feat/nav-v2`
**Источник:** «Trip Share — Карта экранов и аудит навигации» (v2.0) + брейнсторм-сессия.

## 1. Проблема (из аудита)

Текущая навигация смешивает две модели: поездко-центричную (внутри поездки —
Активности/Расходы/Фото/Итоги) и функциональную (глобальные вкладки Расходы/Фото).
Следствия, подтверждённые пользователем:

1. Вкладки «Расходы»/«Фото» всегда видны → кажутся глобальными, но привязаны к
   неявно выбранной «активной» поездке (первой в списке).
2. Непонятно, в какой поездке находишься.
3. Дублирование: разделы есть и в нижнем меню, и внутри поездки.

Сценарий пользователя: **одна активная поездка за раз**.

## 2. Целевая модель

Два уровня навигации с чёткими границами:

- **Нижнее меню — глобальные разделы приложения** (4): `Сегодня · Поездки · Финансы · Профиль`.
- **Контекстные под-табы внутри поездки** (сегментный контрол под названием
  поездки): `План · Расходы · Фото · Участники · Итоги`.

> Уточнение к аудиту: под-таб поездки называем **«Расходы»** (расходы этой
> поездки), а глобальную нижнюю вкладку — **«Финансы»** (кросс-поездочный хаб),
> чтобы одно слово не означало два разных уровня.

«Расходы»/«Фото» перестают быть глобальными вкладками: данные конкретной поездки
живут под `/trip/:id/...` (видно название поездки → контекст явный), а глобальный
раздел **Финансы** становится *настоящим* кросс-поездочным (все расходы/долги по
всем поездкам). Единая кнопка **«+»** убирает разрозненные точки создания.

## 3. Маршрутизация (URL)

Поездочный контекст переносится в URL — это и фикс навигации, и масштабируемость.

```
/                       Сегодня (дашборд: текущая поездка + кросс-поездочное)
/trips                  Список (тулбар: Активные / Будущие / Архив)
/trip/:id               → редирект на /trip/:id/plan
/trip/:id/plan          Активности поездки (бывш. «Активности»)
/trip/:id/expenses      Расходы поездки + вход в баланс
/trip/:id/photos        Фото поездки
/trip/:id/members       Участники + приглашение + редактирование/удаление поездки
/trip/:id/summary       Итоги (Wrapped)
/finance                Глобальные финансы (под-табы: Расходы/Долги/Переводы/Аналитика/Категории)
/profile                Профиль (Настройки/Уведомления/Аккаунт/Поддержка)

# детальные/создание (как сейчас, но создание чаще через «+»):
/activity/:id, /activity/:id/edit, /activity/:id/complete
/expense/:id, /expense/new, /receipt
/balance, /settle
/upload, /bingo, /wrapped, /wrapped/me
/profile/edit, /profile/notifications, /profile/health
```

**«Текущая поездка»** (для Сегодня и таргета «+»): хранится в `localStorage`
(`tm_current_trip`); дефолт — первая активная, иначе первая в списке; при удалении
текущей — сбрасывается на дефолт. Заменяет неявный `useActiveTripId = trips[0]`.

## 4. Экраны и компоненты

### 4.1 Нижнее меню (`BottomNav`)
4 пункта: Сегодня (`/`), Поездки (`/trips`), Финансы (`/finance`), Профиль (`/profile`).
Иконки: 🏠 / 🧳 / 📊 / 👤. Профиль остаётся пунктом меню (не аватар), чтобы
сохранить 4-слотовую симметрию вокруг центральной «+».

### 4.2 Кнопка «+» (`CreateFab`)
Плавающая кнопка по центру над нижним меню. Тап → шит «Создать»:
`🧳 Новая поездка · 💸 Новый расход · 📅 Новая активность · 📷 Загрузить фото`.
Расход/активность/фото таргетят **текущую поездку**; если поездок нет — ведём на
создание поездки.

### 4.3 Сегодня (`/`, дашборд)
Текущая поездка (карточка), мой баланс по ней, **Напоминания** (ближайшие
активности со временем + непогашенные долги), последние расходы, последние
активности. Источник — существующие per-trip эндпоинты текущей поездки + долги.
(«Напоминания» = производное от ближайших активностей и моих долгов, без нового бэка.)

### 4.4 Поездки (`/trips`)
Тулбар **Активные / Будущие / Архив** (уже реализован как Активные/Планируются/
Завершённые — переименовать ярлыки). Карточка → `/trip/:id` → `/trip/:id/plan`.

### 4.5 Оболочка поездки (`TripShell`, `/trip/:id/*`)
Шапка: «← к поездкам», название поездки, статус, ⚙️ (на `/members`). Под шапкой —
сегментный контрол: `План · Финансы · Фото · Участники · Итоги`, синхронизированный
с URL. Контент рендерится дочерним роутом. Это устраняет дубль: внутренние табы
TripDetails = эти под-маршруты (текущий `TripDetails` распиливается на под-экраны).

- **План** — список активностей (как вкладка «Активности» сейчас).
- **Расходы** — расходы поездки + «Баланс →» + «Новый расход».
- **Фото** — галерея + загрузка + Bingo.
- **Участники** — список, «Пригласить», «Изменить поездку», «Удалить» (Опасная зона).
- **Итоги** — Wrapped + статы.

### 4.6 Финансы — глобально (`/finance`)
Под-табы: **Все расходы · Все долги · Все переводы · Аналитика · Категории**.
Кросс-поездочно по всем поездкам пользователя. Требует бэка (раздел 5).

### 4.7 Профиль (`/profile`)
Под-разделы: **Настройки** (тема + нативная валюта — см. §6), **Уведомления**
(есть), **Аккаунт** (данные Telegram, выход), **Поддержка** (ссылка на бота/чат).

## 5. Бэкенд — кросс-поездочные эндпоинты (новое)

Новый контроллер `me` (под JWT, без tripId — агрегирует по членствам пользователя):

```
GET /me/expenses      — все расходы по всем поездкам пользователя (+ tripTitle)
GET /me/debts         — мои долги/мне должны, агрегировано по поездкам
GET /me/settlements   — все зафиксированные переводы пользователя
GET /me/analytics     — суммы: всего, по категориям, по поездкам, по месяцам
```

Все суммы **группируются по валюте** (см. §6): пока без конвертации не смешиваем
разные валюты — каждая агрегатная цифра либо в рамках валюты, либо разбита по
валютам. Категории берём из существующего `expense.category`.

Реализация: переиспользуем `MembershipService` (список поездок юзера) +
существующие репозитории Expense/ExpenseParticipant/Settlement.

## 6. Зависимость: валюта (ОТКРЫТЫЙ ВОПРОС)

Глобальные «Все расходы»/«Аналитика» по поездкам с **разными** `baseCurrency`
требуют либо (а) показа с разбивкой по валютам, либо (б) конвертации в одну
«нативную» валюту пользователя. Полноценная конвертация = отдельная фича (нативная
валюта в настройках + источник курсов FX), которую мы начинали и поставили на паузу.

**Решение для v2.0:** Финансы и аналитика работают с **разбивкой по валютам** (без
FX). Это корректно и не блокирует навигацию. Поле «нативная валюта» добавляем в
Настройки (Профиль) как заготовку для будущей конвертации. Сам FX-механизм
(источник курсов, фиксация курса на расходе через уже существующее
`expense.exchangeRate`) — отдельная спека после v2.0.

## 7. Фазы реализации (внутри «всё v2.0»)

1. **Скелет навигации** (фронт): 4-вкладочный `BottomNav`, `CreateFab`, маршруты
   `/trip/:id/*`, `TripShell` + распил `TripDetails` на под-экраны, `tm_current_trip`,
   переименование тулбара Поездок. Убирает все три боли. Использует текущие эндпоинты.
2. **Глобальные Финансы** (бэк+фронт): эндпоинты `/me/*`, экран `/finance` с
   под-табами Расходы/Долги/Переводы.
3. **Аналитика и Категории**: `/me/analytics`, графики (CSS-бары, без либ),
   разбивка по категориям/месяцам/поездкам.
4. **Профиль и Сегодня-полиш**: Аккаунт/Поддержка, нативная валюта в Настройках,
   блок «Напоминания» на Сегодня.

Каждая фаза — отдельный PR на ветке `feat/nav-v2`, деплой по готовности.
