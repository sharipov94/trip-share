# Frontend Technical Specification

## Платформа

Telegram Mini App.

## Технологии

- **React** + **TypeScript**
- **Telegram SDK** (`@telegram-apps/sdk`) — initData, theme params, haptics, BackButton
- **TanStack Query** — серверное состояние, кеш, оффлайн-persist
- **Zustand** — локальное UI-состояние
- **Tailwind** — стилизация (с учётом Telegram theme params для тёмной/светлой темы)

## Темизация (выбор палитры)

Пользователь сам выбирает оформление — это фича продукта, не просто тёмная/светлая
тема. Живой референс: [`../design/home-palettes.html`](../design/home-palettes.html).

**Палитры (4 + авто):**

| Тема | Тип | Характер |
|------|-----|----------|
| `sunset` (по умолчанию) | тёмная | закатные градиенты, тёплая, ностальгическая |
| `neon` | тёмная | электрик-бирюза/фиолет, киберпанк |
| `pastel` | **светлая** | мягкие розово-голубые, нежная |
| `acid` | тёмная | ядовитый лайм/магента, дерзкая |
| `auto` | — | следует системной теме Telegram: светлая → `pastel`, тёмная → `sunset` |

**Реализация:**
- Палитра — это набор **CSS-переменных** (`--bg`, `--ink`, `--g1`, `--g-debt` …),
  переключается атрибутом `data-theme` на корне. Один набор разметки, разные токены.
- **Источник истины — сервер:** `users.theme` (см. [03](03-database-schema.md)),
  читается из `GET /users/me`, меняется `PATCH /users/me`.
- **Мгновенное применение:** выбранная тема дублируется в Telegram CloudStorage /
  localStorage и применяется до загрузки профиля, чтобы не было «вспышки» дефолтной
  темы при старте.
- Базовые цвета фона мягко согласуются с Telegram theme params, но палитра
  доминирует над ними (это намеренно брендовый, а не системный вид).

## Навигация

Bottom Navigation: **Home · Trips · Expenses · Memories · Profile**.

Использовать нативную Telegram `BackButton` для возврата внутри стеков.

## Авторизация и онбординг

Регистрация **не требуется** — при первом открытии берём `initData` из Telegram SDK,
шлём на `POST /auth/telegram`, получаем JWT. Имя/аватар подставляются из Telegram.
Никаких форм входа. Детали — [07-auth](07-auth.md).

## Приглашение участников (контакты Telegram)

Mini App **не читает список контактов** — приглашение идёт через нативный share Telegram:

1. На экране поездки кнопка «Пригласить» → `POST /trips/:id/invite` возвращает deep-link.
2. Открываем нативный share (`tg://msg_url` / Telegram SDK share) с этой ссылкой — пользователь
   сам выбирает контакты.
3. При открытии Mini App по такой ссылке читаем `start_param` и вызываем
   `POST /trips/:id/join`.

> Добавление из контактов телефона появится в отдельном мобильном приложении (будущая фаза).

## Экраны

### Home («Сегодня»)
Содержит: сегодняшние активности, ближайшие события, долги, уведомления, новые фото.

### Trips
Список путешествий. Карточка: название, даты, участники, ближайшая активность.

### Trip Details
Tabs: **Overview · Activities · Expenses · Memories · Summary**.

- **Activities** — создание активности, голосование (Иду/Не иду), комментарии,
  просмотр участников.
- **Expenses** — создание расхода, загрузка чека, просмотр долгов и плана переводов
  (данные из `GET /trips/:id/balance`). У каждого `pending`-долга — кнопка
  **«Перевести»** (диплинк в чат с реквизитами, см.
  [05](05-expenses-and-settlements.md#закрытие-долга-в-один-тап)).
- **Memories** — загрузка фотографий, просмотр, таймлайн путешествия. Отдельный вид
  **«Момент»** — карусель/коллаж кадров одного события от разных авторов
  ([«N ракурсов»](01-product-concept.md#один-момент--n-ракурсов-ключевой-артефакт)).
- **Summary** — статистика, достижения, лучшие фото, история поездки. Подаётся
  в **Wrapped-формате**: полноэкранная вертикальная история карточек-слайдов (одна
  яркая цифра на экран), групповой и персональный варианты, кнопка «поделиться».

### Геймификация (UI)
- **Photo-prompt** приходит как карточка с действием «снять сейчас»; первый
  откликнувшийся получает бейдж (haptic + анимация).
- **Bingo поездки** — сетка заданий, ячейки закрываются загруженными фото.
- **Титулы-роли** показываются в персональном Wrapped по итогам поездки.

### Profile
Имя, аватар, реквизиты для переводов, **оформление (выбор палитры)**, настройки
уведомлений, настройки здоровья (opt-in).

## Реалтайм

Подписка на WebSocket-комнату `trip:{id}` при открытии Trip Details. Входящие
события (`activity.created`, `vote.updated`, `expense.created`, `balance.updated`,
`memory.added`) инвалидируют соответствующие TanStack Query-ключи, чтобы UI группы
обновлялся синхронно. Fallback при отсутствии сокета — `refetchInterval` (polling).

## Offline Support

**Чтение.** Кеширование поездок, активностей, расходов через
`@tanstack/query-persist-client` (хранилище — Telegram CloudStorage / IndexedDB).

**Запись.** Оффлайн-мутации не пишутся напрямую — ставятся в очередь оптимистичных
мутаций (TanStack `onMutate` + персистентная очередь). При восстановлении сети:

- каждая отложенная мутация отправляется с собственным `Idempotency-Key`
  (см. [05](05-expenses-and-settlements.md#идемпотентность)), чтобы повторная
  отправка не создала дубль расхода;
- конфликты (расход изменён другим участником) разрешаются по правилу
  «сервер — источник истины»: при `409` локальная оптимистичная запись
  откатывается и пользователю показывается актуальное состояние.

## UX-детали

- Учитывать Telegram theme params (фон, акцентный цвет) — не хардкодить палитру.
- Haptic feedback на ключевые действия (голос, создание расхода).
- MainButton использовать для подтверждающего действия на экранах-формах.
